## Overview

Seeders populate the database with demo data for development and testing. They create sample companies, users, customers, inventory, and transactions.

<Warning>
Never run seeders in production unless you specifically want demo data. Seeders are for development and testing only.
</Warning>

## Running Seeders

### Run All Seeders

```bash
php artisan db:seed
```

### Run Specific Seeder

```bash
php artisan db:seed --class=CompanySeeder
```

### Fresh Migration with Seeding

```bash
php artisan migrate:fresh --seed
```

## Seeder Execution Order

Seeders run in the following order (defined in `DatabaseSeeder.php`):

1. **CompanySeeder** - Creates demo companies
2. **UserSeeder** - Creates test users
3. **SubscriptionSeeder** - Creates company subscriptions
4. **OperationalDataSeeder** - Creates customers, equipment, and orders
5. **InventorySeeder** - Creates inventory items
6. **BillingSeeder** - Creates billing documents

## Seeder Files

All seeders are located in `database/seeders/`.

### DatabaseSeeder.php

Main seeder that orchestrates all other seeders.

**File**: `database/seeders/DatabaseSeeder.php`

**Execution Order**:
```php
$this->call([
    CompanySeeder::class,
    UserSeeder::class,
    SubscriptionSeeder::class,
    OperationalDataSeeder::class,
    InventorySeeder::class,
    BillingSeeder::class,
]);
```

## Seeder Features

### Idempotency

All seeders use `updateOrCreate()` instead of `create()`, making them safe to run multiple times:

```php
Company::query()->updateOrCreate(
    ['name' => 'ElectroFix Cliente Demo'], // Find by
    [...] // Update or insert with this data
);
```

### Relationships

Seeders maintain referential integrity:
- Users belong to companies
- Customers belong to companies
- Equipment belongs to customers and companies
- Orders reference customers, equipment, and companies
- Inventory items belong to companies
- Billing documents reference users, customers, and companies

### Multi-Tenancy Testing

Two separate companies allow testing:
- Data isolation
- Cross-tenant query protection
- Company-specific features
- Subscription plan differences

### Real-World Scenarios

Demo data includes:
- Low stock alerts (items below threshold)
- Different order statuses (diagnostic, quote)
- AI diagnostics with JSON data
- Mixed billing (services + products)
- Tax calculations
- Multi-currency support (MXN)

## Resetting Demo Data

To reset to fresh demo data:

```bash
php artisan migrate:fresh --seed
```

<Warning>
This will DROP all tables and recreate them with fresh demo data. All existing data will be lost.
</Warning>

## Custom Seeding

To create your own seeder:

```bash
php artisan make:seeder YourSeeder
```

Add to `DatabaseSeeder.php`:

```php
$this->call([
    // ... existing seeders
    YourSeeder::class,
]);
```
